# 海报工具第二期人工验收与真实Offer模块修复设计

## 背景

在第二期子项目 B 中，海报工具已经完成以下关键工作：

- 正式导出顶部共享几何修复
- 图片数量上限从 1 到 3 张扩容为 1 到 5 张
- `4-image` 与 `5-image` 的正式布局支持
- 预览 4 / 5 图槽位扩容
- 视觉回归脚本补充了 `four-images-desktop`、`five-images-desktop` 和 `header-safe-zone` 场景

虽然这些开发和自动化回归已经完成，但当前仍存在两个未闭环的问题：

1. 第二期人工验收矩阵还只是模板和局部检查项，尚未形成完整的执行记录。
2. 用户提供的截图显示 `真实Offer` 标签模块在部分场景下存在明显布局问题，包括标签与下方图片或文字内容框重叠，以及预览版式错乱，这意味着在跨浏览器与跨分辨率环境下仍可能存在实际可见的展示异常。

因此，本轮子项目 A 的目标不是继续泛化改造整站，而是围绕“人工验收 + 真实Offer 模块定向修复 + 回归闭环”完成第二期真正可交付的验证工作。

## 目标

本轮子项目的目标如下：

- 在当前环境可执行的近似浏览器矩阵中完成完整的人工验收执行。
- 覆盖以下浏览器基准：
  - Chrome
  - Firefox
  - WebKit（Safari 近似）
  - Chromium（Edge 近似）
- 覆盖以下分辨率：
  - `1920×1080`
  - `1366×768`
  - `375×667`
  - `768×1024`
- 对 `真实Offer` 模块的布局问题进行定向定位与修复。
- 修复后对受影响模块执行跨浏览器、跨分辨率的专项回归验证。
- 将结果、截图证据、问题编号和复验状态回填到验收文档中，形成闭环。

## 非目标

本轮明确不做以下事项：

- 不在本轮展开真实原生 Safari 和真实原生 Edge 浏览器的系统级人工验收，仅使用当前环境可执行的近似基准。
- 不重新设计整个海报模板，不对标题区、路径区、背景区做无关重排。
- 不把所有视觉偏差都纳入本轮修复，只聚焦 `真实Offer` 模块及其直接关联的槽位与标签安全距离问题。
- 不优先处理弱网、暗黑模式、批量导入等与 `真实Offer` 模块无直接因果关系的次级问题，除非在执行本轮验收时发现其与当前故障同根因。

## 问题定义

### 1. 故障 A：真实Offer标签与下方图片 / 文字框重叠

从用户提供的截图看，当前问题并不只是 `offer-chip` 文案位置偏移，而是：

- 标签与下方图片视觉上贴得过近
- 标签与图片内容区或文字区发生遮挡
- 不同分辨率下重叠程度不同

这类问题通常来自以下几类根因：

- `offer-chip` 定位只按静态点位计算，未充分考虑不同图片布局下的安全距离
- 辅图槽位在 `4-image` / `5-image` 布局中下探过深，挤占了标签的安全区域
- 预览端采用 CSS 百分比布局，而正式结果采用导出坐标，导致两者视觉不一致

### 2. 故障 B：真实Offer模块预览版式错乱

当前预览中的问题不是简单“偏了一点”，而是模块结构关系可能已经错位，表现为：

- 右侧图片区内元素相对顺序异常
- 某些断点下 `真实Offer` 标签与图片区元素相互覆盖
- 预览与正式生成图在相同场景下不一致

因此，这一问题需要视为：

- 预览局部布局规则问题
- 右侧图片区在多图状态下的断点适配问题

## 执行方案

### 1. 验收执行策略

本轮采用“自动化辅助取证 + 人工判定落表”的执行方式。

第一层：自动化辅助取证

- 使用浏览器脚本批量跑近似四浏览器基准
- 在 4 个分辨率下采集：
  - 预览截图
  - 正式结果截图
- 自动化只负责采集证据，不直接替代人工判定

第二层：人工判定落表

- 对每组截图检查：
  - `真实Offer` 标签是否压住下方图片或文字框
  - 预览与正式结果是否一致
  - 不同分辨率下是否有错位、溢出、遮挡
- 将结论填写到验收矩阵中，不能只保留脚本日志

### 2. 问题归并策略

为了避免同一根因在多个环境中重复创建问题，本轮默认按根因归并为最多 3 个 issue：

- `ISSUE-A`
  - `真实Offer` 标签与下方图片 / 文字框重叠
- `ISSUE-B`
  - `真实Offer` 模块在预览中的布局错乱
- `ISSUE-C`
  - `4-image` / `5-image` 右侧图片区安全距离不足

如果某个异常可以归入上述 issue，则只在矩阵中引用 issue 编号，不单独拆出冗余问题。

### 3. 问题分级

本轮问题等级定义如下：

- `P1`
  - 阻塞生成或核心内容不可读
  - 例如：标签压住主要内容、图片区文字不可辨认、正式结果错位影响交付
- `P2`
  - 不阻塞主流程，但明显偏离设计规范
  - 例如：平板端错位、移动端局部遮挡、预览与导出不一致
- `P3`
  - 轻微视觉偏差
  - 例如：局部间距不精确、标签位置有轻微偏移但不影响阅读

本轮修复优先级：

- 先清理 `P1`
- 再处理与 `真实Offer` 模块直接相关的 `P2`
- `P3` 记录但不承诺全部在本轮清零

## 风险与注意事项

- 本轮的 Safari / Edge 结果属于近似基准，不应误标为真实原生浏览器人工结论。
- 若 `真实Offer` 模块问题实际来源于共享槽位规则，则允许进行最小范围的公共规则修复，但不得借机扩展为整站重排。
- 若修复后仅在单一近似浏览器中出现轻微偏差，需记录并定级，不应直接触发大规模重构。
- 若验收记录缺失截图路径或 issue 编号，即视为闭环未完成。
