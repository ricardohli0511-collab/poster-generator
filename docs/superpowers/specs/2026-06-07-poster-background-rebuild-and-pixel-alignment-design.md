# 海报背景重创与像素级对位设计

## 背景

当前项目已经完成以下两项基础能力：

- 海报工具支持单个与批量的分类素材上传。
- 预览与导出已统一执行“有素材才显示对应标签”的规则。

但用户反馈的更高优先级问题仍然存在：

1. 当前正式背景仍不符合“香港灣區教育諮詢促進會 HONG KONG GREATER BAY AREA EDUCATIONAL PROMOTION ASSOCIATION”的目标设计要求。
2. 当前预览版式虽然已接近参考样式，但与 `assets/模拟素材.jpg` 仍存在明显视觉偏差，尚未达到像素级还原标准。

本轮工作的目标不是继续做局部修补，而是建立一套可以长期复用的视觉基准体系：先重创符合要求的专属背景母版，再将预览与导出共同收敛到统一的高精度坐标数据库。

## 目标

本轮设计目标如下：

- 以“协会官方识别元素 + 模拟素材构图语言融合重创”的方式生成新的专属背景母版。
- 对 `assets/模拟素材.jpg` 建立完整的页面元素参考数据库，用于约束所有前景组件的精确位置与尺寸。
- 将当前预览版式与正式 SVG 导出统一到同一套基准数据，避免预览与导出偏差。
- 把对位标准从“近似还原”升级为“像素级还原”，并通过 100%、50%、200% 三档做对照校验。

## 非目标

本轮不做以下扩展：

- 不修改单个/批量分类上传的业务交互规则。
- 不引入第三方图片生成 API 或远程模板平台。
- 不将用户上传的案例图当作正式背景使用。
- 不改造为全新的 Canvas 或全新渲染技术栈，仍在现有 HTML/CSS + SVG 导出框架内完成。

## 视觉基准来源

### 1. 协会品牌识别来源

`assets/香港灣區教育諮詢促進會 HONG KONG GREATER BAY AREA EDUCATIONAL PROMOTION ASSOCIATION.png` 继续作为品牌识别来源，保留以下要素：

- 协会 Logo
- 协会中英文名称
- 蓝金品牌气质
- 夜景城市与上升动势的品牌语义

### 2. 构图与节奏来源

`assets/模拟素材.jpg` 作为本轮成图构图的主要参考，重点吸收以下维度：

- 标题牌匾的横向比例与承托结构
- 顶部 slogan 与标题区的纵向关系
- 左侧三段路径卡的节奏、间距与箭头过渡
- 右侧图片区的主次关系
- 底部城市、金色箭头、金带装饰与 footer 的整体视觉收束

### 3. 融合原则

本轮不做“简单替换背景”，而是执行“融合重创”：

- 协会品牌识别优先保留。
- 模拟素材的构图、密度、动势与层次优先还原。
- 新背景母版应在视觉上看起来属于协会专属设计，而不是简单拷贝参考图。

## 核心方案

### 1. 新专属背景母版

新增一张新的固定背景母版，作为正式唯一背景源。

该背景需满足：

- 顶部保留协会 Logo 与机构中英文识别。
- 背景蓝色层次、金色装饰、底部城市与上升动势需向 `模拟素材.jpg` 对齐。
- 中部需预留标题牌匾、路径区和图片区的安全视觉空间。
- 预览与正式 SVG 导出都只允许使用这张背景母版。

原有背景图不再作为“最终设计成品背景”直接使用，而是退回到品牌识别参考素材角色。

### 2. 参考基准数据库

新增一份基于 `1080 x 1920` 坐标系的参考数据库，作为本轮所有视觉修正的唯一真值来源。

数据库至少覆盖以下模块：

- 品牌顶部区
- slogan 区
- 标题牌匾区
- 路径标题行
- 学期 pill
- 高中阶段卡片
- 副学士阶段卡片
- 学士阶段卡片
- 右侧成绩单 / 证书主图区
- 右侧 offer 区
- 底部城市与 footer 区

每个模块记录以下参数：

- `x`
- `y`
- `width`
- `height`
- `marginTop`
- `marginRight`
- `marginBottom`
- `marginLeft`
- `padding`
- `fontSize`
- `fontWeight`
- `lineHeight`
- `letterSpacing`
- `borderWidth`
- `borderRadius`
- `shadow`
- `rotation`

### 3. 误差控制原则

用户要求中提到“提取数据误差控制在 0.5 像素以内”。在浏览器/CSS/SVG 的实际实现语境下，本轮设计采用以下执行口径：

- 参考数据记录尽量细化到小数级，允许使用小数像素值。
- 实施与校验时，以 `1080 x 1920` 母版坐标系为准，将关键模块控制在 `1px` 以内的可见偏差。
- 对于阴影、发光、纹理与背景渐变这类非刚性视觉层，不以机械像素点逐点一致为目标，而以肉眼不可感知偏差为准。

这一定义是为保证实现可落地，同时仍然满足“像素级还原”的目标。

## 文件职责与改动范围

### 1. `assets/`

新增新的专属背景成品文件。

职责：

- 保存本轮融合重创后的唯一固定背景母版。
- 不承载用户上传素材内容。

### 2. `template-config.example.js`

职责调整：

- 更新正式背景路径，切换到新的背景母版。
- 按新的参考数据库同步调整 `textBlocks` 与 `imageLayouts`。

### 3. `app.js`

职责调整：

- 将导出 SVG 的关键坐标、尺寸和文本参数切换到新的基准数据。
- 保持素材分类上传、标签显隐与背景强约束不回退。
- 如有必要，将当前零散的固定参数收束成更清晰的布局配置结构。

### 4. `styles.css`

职责调整：

- 按参考数据库重算预览层所有关键元素的尺寸、位置、间距、字距、字重和视觉效果。
- 重点修正标题牌匾、路径卡、右侧图片区、footer 以及背景覆盖层的关系。
- 尽量减少影响像素对位的弹性布局与模糊缩放行为。

### 5. `index.html`

职责调整：

- 保持现有预览 DOM 结构为主。
- 仅在必要时补充包裹层或结构钩子，支撑更稳定的对位控制。

### 6. `app.test.js`

职责调整：

- 保持现有分类上传回归测试。
- 补充新背景路径与导出结构的关键回归项，防止背景源回退。

## 实施策略

### 阶段一：背景重创

先完成背景母版的更新，解决“底图不对”的根问题。

执行要点：

- 以协会品牌识别为上层规范。
- 以 `模拟素材.jpg` 为构图参考重做底图节奏。
- 导出和预览统一引用新的背景文件。

### 阶段二：参考数据库建立

把参考图中的核心元素拆成模块并记录坐标。

执行要点：

- 使用 `1080 x 1920` 作为唯一母版尺寸。
- 对所有关键模块建立坐标与样式参数表。
- 数据库中每个值都要服务于预览与导出的共同校准。

### 阶段三：预览层对位

在 `styles.css` 中逐模块校准。

优先顺序：

- 顶部品牌识别区
- 标题牌匾
- 左侧路径区
- 右侧图片区
- 底部收束区

### 阶段四：导出层同步

在 `app.js` 中将 SVG 输出同步到相同基准。

执行要点：

- 不允许只改预览不改导出。
- 右侧图片布局、chip、文本和 footer 的位置都要与预览同步。

## 重点核查清单

本轮必须专项核查以下维度：

- 网格基线对齐度
- 各模块的外边距和内边距
- 图文组件的相对位置关系
- 标题牌匾比例与装饰完整度
- 右侧图片区块的主次关系
- 底部视觉重心与城市层次
- 蓝金配色的饱和度、明度与金色质感
- 文字的字重、字距、行高与描边/阴影感知

## 测试与验证

### 1. 自动化测试

自动化测试继续承担“结构正确性”与“规则不回退”的职责。

至少保留以下验证：

- 分类上传逻辑仍可用。
- 条件渲染标签仍准确。
- 正式输出必须依赖固定背景配置。
- 导出结构中使用的是新背景母版而非旧背景路径。

### 2. 手工视觉校验

手工校验用于完成“像素级视觉还原”的最终验收。

校验方式：

- 100% 缩放：核对真实对位关系。
- 50% 缩放：核对整体节奏、视觉重心与留白。
- 200% 缩放：核对边框、字距、模块边距和细部错位。

每次校验都需覆盖：

- 标题区
- 路径区
- 图片区
- footer 区
- 背景色调与装饰层

## 验收标准

满足以下条件可视为本轮完成：

- 新背景母版已完成，且符合“协会官方识别 + 模拟素材构图”的融合要求。
- 当前预览与正式 SVG 导出统一使用新的背景母版。
- 参考数据库已建立，并成为预览与导出对位的共同基准。
- 标题牌匾、路径卡、右侧图片区与 footer 等核心区域完成像素级对位收敛。
- 在 100%、50%、200% 三档缩放下均未发现明显偏移、错位或色彩失真。
- 分类上传、标签显隐与固定背景约束不发生回退。

## 风险与注意事项

- 浏览器 CSS 与 SVG 在字体渲染上的细微差异可能造成极小的视觉偏差，因此必须把重点放在“整体观感无偏差”和“关键模块在 1px 以内”。
- 若当前 DOM 结构对标题牌匾或图片区的精确控制不够，需要允许小幅结构调整，但应避免破坏现有上传功能。
- 背景母版属于本轮成败关键，若背景层不先重构，单独微调前景很难真正达到参考图标准。
